    **ViewModel class:**

    `@HiltViewModel class ActivityFeedViewModel @Inject constructor(private val supabase: SupabaseClient, private val authRepository: AuthRepository) : ViewModel()`

    `private val _uiState = MutableStateFlow(ActivityFeedUiState()); val uiState: StateFlow<ActivityFeedUiState> = _uiState.asStateFlow()`

    `init { loadFeed() }`

    `fun loadFeed()` — `viewModelScope.launch { ... }`:
    1. `_uiState.value = _uiState.value.copy(isLoading = true, error = null)`
    2. Compute `sevenDaysAgo`: `java.time.Instant.now().minus(7, java.time.temporal.ChronoUnit.DAYS).toString()` — gives ISO-8601 UTC string compatible with Supabase TIMESTAMPTZ filter.
    3. Query:
    ```kotlin
    val events = supabase.from("activity_events")
        .select("*, users(id, username)") {
            filter {
                gte("created_at", sevenDaysAgo)
            }
            order("created_at", Order.DESCENDING)
            limit(50L)
        }
        .decodeList<ActivityEventItem>()
    ```
    Note: The `*, users(id, username)` foreign table embed syntax works because `activity_events.user_id` references `public.users(id)`. The RLS policy `activity_events_select_following` (from 05-01) automatically filters to own + followed users' events — no additional filter needed client-side.
    4. `_uiState.value = ActivityFeedUiState(isLoading = false, events = events)`
    5. Wrap the entire block in try/catch; on exception: `_uiState.value = ActivityFeedUiState(isLoading = false, error = "Không thể tải hoạt động bạn bè. Kiểm tra kết nối mạng.")`

    Import `io.github.jan.supabase.postgrest.from`, `io.github.jan.supabase.postgrest.query.Order`, `java.time.Instant`, `java.time.temporal.ChronoUnit`.

    Add `fun retry() { loadFeed() }` as a convenience method for the error state retry button.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → activity_events SELECT | RLS policy (05-01) scopes results to own + followed users; client cannot bypass |
| client → users embedded SELECT | Foreign table embed via Supabase; same RLS as users table (public SELECT per 05-01) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | Information Disclosure | activity_events SELECT (friend scoping) | mitigate | RLS `activity_events_select_following` enforced server-side via EXISTS subquery on friendships; client cannot widen the filter |
| T-05-04-02 | Information Disclosure | metadata JSONB exposed to client | accept | metadata contains only challenge_date (date, not time) and elo_earned (integer) — no PII |
| T-05-04-03 | Denial of Service | limit on feed query | mitigate | limit(50L) + gte("created_at", sevenDaysAgo) caps result set size |
| T-05-04-04 | Spoofing | formatRelativeTime parse failure (malformed timestamp) | mitigate | try/catch returns "" on Instant.parse failure; card still renders without crash |
| T-05-04-05 | Information Disclosure | embedded users(id, username) in feed events | accept | Only id + username returned; consistent with public search exposure in SOCIAL-01 |
</threat_model>
