    `fun loadProfile(targetUserId: String)` — sets `currentTargetUserId = targetUserId`, launches `viewModelScope.launch { ... }`:
    1. Determine `isCurrentUser = (authRepository.currentUserId == targetUserId)`
    2. Launch two parallel async blocks (use `coroutineScope { val leaderDeferred = async { }; val progressDeferred = async { } }`):
       - `leaderDeferred`: `supabase.from("leaderboard").select { filter { eq("user_id", targetUserId) } }.decodeSingleOrNull<LeaderboardEntry>()` (reuse existing `LeaderboardEntry` from LeaderboardRepository)
       - `progressDeferred`: `supabase.from("user_progress").select("streak_count,total_games") { filter { eq("user_id", targetUserId) } }.decodeSingleOrNull<UserProgressPublic>()`
    3. Import `kotlinx.coroutines.async` and `kotlinx.coroutines.coroutineScope`
    4. Await both. If `leaderEntry == null`, emit error "Không tìm thấy người dùng này."
    5. Build `PublicProfileData` merging the two results (progress fallback: streakCount=0, totalGames=0 if null)
    6. If not `isCurrentUser`, call private `checkFollowStatus(targetUserId)` — returns Boolean: `supabase.from("friendships").select("follower_id") { filter { eq("follower_id", currentUserId); eq("following_id", targetUserId) }; limit(1L) }.decodeSingleOrNull<Any>() != null`
    7. Emit `PublicProfileUiState(isLoading=false, profile=..., isFollowing=..., isCurrentUser=...)`
    8. Wrap in try/catch; on exception emit `uiState.copy(isLoading=false, error="Không thể tải hồ sơ. Kiểm tra kết nối mạng.")`

    `fun toggleFollow()` — launches `viewModelScope.launch { ... }`:
    - `val targetId = currentTargetUserId ?: return@launch`
    - `val currentUserId = authRepository.currentUserId ?: return@launch`
    - If `_uiState.value.isFollowing`:
      - `supabase.from("friendships").delete { filter { eq("follower_id", currentUserId); eq("following_id", targetId) } }`
      - Optimistically update: `_uiState.value = _uiState.value.copy(isFollowing = false)`
    - Else:
      - `supabase.from("friendships").insert(buildJsonObject { put("follower_id", currentUserId); put("following_id", targetId) })`
      - Optimistically update: `_uiState.value = _uiState.value.copy(isFollowing = true)`
    - Wrap in try/catch; on exception revert the optimistic update and set error.

    Import `io.github.jan.supabase.postgrest.from`, `io.github.jan.supabase.postgrest.query.filter.FilterOperator`, `kotlinx.serialization.json.buildJsonObject`, `kotlinx.serialization.json.put`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Supabase leaderboard | Public SELECT — any authenticated user reads leaderboard rows; no writes from this screen |
| client → Supabase friendships | INSERT/DELETE gated by RLS: `follower_id = auth.uid()` enforced server-side |
| client → Supabase user_progress | Public SELECT per 007 migration; no writes from this screen |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Spoofing | friendships INSERT (follow) | mitigate | RLS `WITH CHECK (follower_id = auth.uid())` prevents impersonation; server validates |
| T-05-03-02 | Tampering | friendships DELETE (unfollow) | mitigate | RLS `USING (follower_id = auth.uid())` prevents deleting others' follows |
| T-05-03-03 | Information Disclosure | user_progress public SELECT | accept | streak_count, total_games are gamification stats; no PII; academic project scope |
| T-05-03-04 | Information Disclosure | leaderboard public SELECT | accept | Already public per 001 migration; consistent with existing design |
| T-05-03-05 | Denial of Service | loadProfile() called on every LaunchedEffect recompose | mitigate | LaunchedEffect(userId) key ensures load only triggers on userId change, not every recompose |
</threat_model>
