Create app/src/main/java/com/example/vietforces/data/fcm/FCMTokenManager.kt:

Package: com.example.vietforces.data.fcm

Imports: com.google.firebase.messaging.FirebaseMessaging, io.github.jan.supabase.SupabaseClient, io.github.jan.supabase.postgrest.from, kotlinx.coroutines.tasks.await, kotlinx.serialization.Serializable, kotlinx.serialization.SerialName, javax.inject.Inject, javax.inject.Singleton

@Serializable data class FcmTokenDto(@SerialName("user_id") val userId: String, @SerialName("token") val token: String)

@Singleton class FCMTokenManager @Inject constructor(private val supabase: SupabaseClient):

companion object { var instance: FCMTokenManager? = null }
init { instance = this }

suspend fun registerToken(userId: String):
  1. val token = FirebaseMessaging.getInstance().token.await()
  2. supabase.from("fcm_tokens").upsert(FcmTokenDto(userId = userId, token = token)) { onConflict = "user_id" }
  3. Wrap entire method in try/catch, log failures silently (do not crash app if FCM unavailable)

suspend fun updateToken(newToken: String, userId: String):
  1. Upsert FcmTokenDto(userId = userId, token = newToken) to fcm_tokens with onConflict = "user_id"
  2. Try/catch with silent failure

suspend fun deleteToken(userId: String):
  1. supabase.from("fcm_tokens").delete { filter { eq("user_id", userId) } }
  2. Try/catch with silent failure

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FCM server→device | FCM message received by VietForcesFirebaseMessagingService; data payload is untrusted |
| notification tap→MainActivity | NAVIGATE_TO intent extra could be manipulated by malicious apps via PendingIntent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Tampering | FCM data payload screen value | mitigate | onMessageReceived only routes to known screen constants ("daily_challenge"); unknown values are ignored — no arbitrary navigation |
| T-04-12 | Spoofing | FCM token in fcm_tokens | accept | Tokens are upserted by authenticated user only (RLS fcm_insert_own + fcm_update_own) — a compromised token allows receiving notifications but not data exfiltration |
| T-04-13 | Information Disclosure | google-services.json API key | mitigate | File must be in .gitignore; it contains FCM sender ID which is low-sensitivity; private key never stored in this file |
| T-04-14 | Information Disclosure | FIREBASE_SERVICE_ACCOUNT_JSON Supabase secret | mitigate | Stored via supabase secrets set; never in source control; used only in send-streak-reminder edge function |
| T-04-15 | Elevation of Privilege | POST_NOTIFICATIONS permission bypass | accept | Android 13+ framework enforces the permission; app requests at runtime per NOTIF-05; notifications silently dropped if denied |
| T-04-16 | Denial of Service | notification spam | accept | FCM rate limits + pg_cron runs once at 19:00 UTC; individual users receive at most 1 notification per day |
| T-04-SC | Tampering | firebase-messaging-ktx npm/gradle install | mitigate | Package from Google's official Maven repo (google() in repositories); version 33.7.0 via Firebase BOM; no additional npm installs |
</threat_model>
