<objective>
Remove the OpenAI API key from the Android APK by routing all AI calls through a Supabase
Edge Function proxy. The Edge Function (Deno/TypeScript) receives the standard OpenAI chat
completions payload from the Android client and forwards it to OpenAI using a server-side key
stored in Supabase secrets.

Purpose: Fulfil FOUND-03 — the OpenAI API key must be absent from the built APK. The key
lives only in Supabase environment variables. AiManager and all existing AI features continue
to work without any change to their call sites.

Output: supabase/functions/openai-proxy/index.ts (thin proxy, no business logic) and an
updated OpenAiClient.kt that targets the proxy URL using SUPABASE_ANON_KEY for authorization.
</objective>

  <files>supabase/functions/openai-proxy/index.ts</files>
  <action>
    Create the directory supabase/functions/openai-proxy/ and the file index.ts inside it.
    This is a Deno TypeScript file for Supabase Edge Functions runtime.

    The function must:

    1. Export a default handler via Deno.serve(async (req) => { ... }).

    2. Handle CORS preflight: if req.method === "OPTIONS", return a Response with status 204
       and headers Access-Control-Allow-Origin: *, Access-Control-Allow-Methods: POST OPTIONS,
       Access-Control-Allow-Headers: authorization content-type x-client-info apikey.

    3. For all other requests: read the OpenAI API key from Deno environment using
       Deno.env.get("OPENAI_API_KEY"). If the key is missing or empty, return a JSON response
       with status 500 and body { error: "OPENAI_API_KEY not configured" }.

    4. Forward the request to OpenAI: fetch the body as text from req body (await req.text()),
       then POST to https://api.openai.com/v1/chat/completions with headers:
         Authorization: Bearer ${openaiKey}
         Content-Type: application/json

    5. Return the OpenAI response to the caller verbatim: use the same status code and return
       the response body as-is. Include the CORS header Access-Control-Allow-Origin: * on the
       returned response.

    6. Wrap the entire handler body in a try/catch; on error return status 500 with
       { error: error.message }.

    This function is intentionally a thin proxy — no prompt validation, no rate limiting,
    no model override logic. Business rules stay in the Android client (AiManager.kt). Keep
    the file under 60 lines.

    Do NOT import any external Deno modules (no npm: or https: imports needed for basic fetch).
    The Deno runtime provides fetch globally. Do not add a deno.json or import map — Edge
    Functions do not need one for stdlib-free code.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Android client → Edge Function | Anon key in Authorization header; Supabase gateway validates JWT |
| Edge Function → OpenAI API | OPENAI_API_KEY in Deno env (server-side only, never sent to client) |
| Edge Function → caller | OpenAI response forwarded verbatim; no secrets in response body |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | OPENAI_API_KEY exposure | mitigate | Key stored only in Supabase secrets (Deno.env); empty string in APK BuildConfig; grep for non-empty OPENAI_API_KEY in APK strings.xml must return 0 hits |
| T-03-02 | Elevation of Privilege | Unauthenticated Edge Function calls | accept | Supabase gateway enforces anon key check; rate limiting is inherent in Supabase free tier; full auth enforcement deferred to Phase 2 when JWT user tokens are available |
| T-03-03 | Tampering | Prompt injection via request body forwarding | accept | The proxy forwards body verbatim — prompt injection is an application-layer risk handled by AiManager system prompts, not a proxy concern |
| T-03-04 | Denial of Service | Unbounded OpenAI API calls through proxy | accept | Supabase Edge Function invocation limits (free tier: 500k invocations/month) provide natural rate limiting; explicit rate limiting deferred to Phase 6 |
| T-03-SC | Tampering | Deno import map / external module supply chain | mitigate | index.ts uses zero external imports (no npm:, no https: imports); only Deno built-in globals (fetch, Deno.env, Deno.serve) — no supply chain surface |
</threat_model>
