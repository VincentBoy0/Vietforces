## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| APK → Supabase REST API | SUPABASE_ANON_KEY travels in APK and over HTTPS |
| local.properties → BuildConfig | Secrets baked into APK at build time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | BuildConfig.SUPABASE_ANON_KEY in APK | accept | Anon key is designed to be public; Supabase RLS (Plan 01-02) enforces row-level access — no sensitive operation is possible with anon key alone |
| T-01-02 | Information Disclosure | local.properties secrets | accept | local.properties is git-ignored; secrets never committed to VCS |
| T-01-03 | Tampering | Hilt-generated component wiring | accept | Generated code is deterministic from annotations; no runtime-mutable injection paths |
| T-01-SC | Tampering | Maven/Gradle dependency resolution | mitigate | Pin all versions via BOM + explicit version.ref in libs.versions.toml; no dynamic version ranges (+); verify with ./gradlew dependencies before merging |
</threat_model>
