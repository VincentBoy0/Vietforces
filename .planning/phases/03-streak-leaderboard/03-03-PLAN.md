    Create `LeaderboardViewModel.kt` in `com.example.vietforces.ui.viewmodel`:

    Define `data class LeaderboardUiState(val isLoading: Boolean = true, val entries: List<LeaderboardEntry> = emptyList(), val myEntry: LeaderboardEntry? = null, val myRank: Int = -1, val selectedTab: LeaderboardTab = LeaderboardTab.ALL_TIME, val error: String? = null)`.

    Annotate `@HiltViewModel class LeaderboardViewModel @Inject constructor(private val repository: LeaderboardRepository, private val supabase: SupabaseClient) : ViewModel()`.

    Private mutable state: `private val _uiState = MutableStateFlow(LeaderboardUiState())`. Public val: `val uiState: StateFlow<LeaderboardUiState> = _uiState.asStateFlow()`.

    Private `var realtimeChannel: RealtimeChannel? = null`.

    In `init {}`: call `loadLeaderboard()` and `subscribeRealtime()`.

    Method `fun selectTab(tab: LeaderboardTab)`: update `_uiState.value = _uiState.value.copy(selectedTab = tab, isLoading = true)`; call `loadLeaderboard()`.

    Private `fun loadLeaderboard()` — launches in `viewModelScope`:
    - Call `repository.getTop50(_uiState.value.selectedTab)` and `repository.getMyEntry()` in parallel using two separate `async` coroutines (or sequentially for simplicity if parallel adds complexity).
    - Compute `myRank`: find index of myEntry in the top50 list (1-based); if not found, set myRank = -1 to indicate "outside top 50".
    - Update `_uiState.value = _uiState.value.copy(isLoading = false, entries = top50, myEntry = myEntry, myRank = myRank, error = null)` on success.
    - On failure: `_uiState.value = _uiState.value.copy(isLoading = false, error = e.message)`.

    Private `fun subscribeRealtime()` — launches in `viewModelScope`:
    - `realtimeChannel = supabase.channel("public-leaderboard")`.
    - Collect two flows: `realtimeChannel!!.postgresChangeFlow<PostgresAction.Insert>(schema = "public") { table = "leaderboard" }` and `postgresChangeFlow<PostgresAction.Update>(schema = "public") { table = "leaderboard" }`.
    - For each change: call `loadLeaderboard()` to refresh the list (simple approach — no in-place mutation needed for top-50 display).
    - Merge both flows: use `merge(insertFlow, updateFlow).onEach { loadLeaderboard() }.launchIn(viewModelScope)`.
    - `realtimeChannel!!.subscribe()`.

    Override `onCleared()`:
    - `viewModelScope.launch { realtimeChannel?.let { supabase.realtime.removeChannel(it) } }`.

    Imports: `io.github.jan.supabase.realtime.channel`, `io.github.jan.supabase.realtime.postgresChangeFlow`, `io.github.jan.supabase.realtime.PostgresAction`, `io.github.jan.supabase.realtime.RealtimeChannel`, `kotlinx.coroutines.flow.merge`, `kotlinx.coroutines.flow.onEach`, `kotlinx.coroutines.flow.launchIn`, `dagger.hilt.android.lifecycle.HiltViewModel`.
