## Testing Gaps

### High Priority ‚Äî Missing Unit Tests (Android)

1. **ViewModel state machine logic** ‚Äî all `sealed class UiState` transitions are untested. Each ViewModel (`AuthViewModel`, `DailyChallengeViewModel`, `LeaderboardViewModel`, etc.) should have unit tests using `kotlinx-coroutines-test` + Turbine.

2. **`AuthViewModel.toFriendlyMessage()`** ‚Äî pure function with 8 branches mapping exception messages to Vietnamese strings. Directly testable with JUnit4 and no mocking.

3. **`DailyChallengeViewModel.computeSecondsUntilMidnightUtc()`** ‚Äî pure time calculation. Should be tested at midnight boundary conditions.

4. **Repository layer** ‚Äî all repositories call Supabase; these need mocking (`MockK` recommended). No mock infrastructure currently exists.

5. **`PreferencesManager`** ‚Äî 570-line SharedPreferences wrapper; no tests for data persistence or defaults.

### High Priority ‚Äî Missing Web Tests

1. **Server actions** ‚Äî `createWord`, `updateWord`, `deleteWord` in `web-admin/src/lib/actions/vocabulary.ts` have complex logic (file upload, path extraction). Integration or unit tests with a mocked Supabase client would catch regressions.

2. **Auth middleware** ‚Äî `web-admin/middleware.ts` enforces session-based access control; no test verifies the redirect logic.

### Medium Priority ‚Äî Missing Backend Tests

1. **Edge function integration tests** ‚Äî `generate-daily-challenge` idempotency and `CRON_SECRET` auth check should be verified with Deno's test runner (`Deno.test`).

2. **RLS policy tests** ‚Äî Supabase supports `pgTAP` for testing RLS policies directly in SQL. None are written. Critical for `users_select_own`, `progress_insert_own`, and the admin role bypass.

3. **Database function tests** ‚Äî `calculate_elo`, `update_streak`, and other PL/pgSQL functions in migrations `002`‚Äì`003` are untested.

### Low Priority

1. **Landing page** (`web-landing/`) ‚Äî static marketing page with no logic; visual regression tests (Playwright/Storybook) would be nice-to-have but are not urgent.

2. **Compose UI tests** ‚Äî Screen-level Compose UI tests (login flow, game flow) using the available `androidx.compose.ui.test.junit4` dependency are not written.
