# Testing Patterns

**Analysis Date:** 2026-07-22

## Test Framework

**Runner:**
- JUnit 4 (`junit:junit:4.13.2`) for local unit tests
- AndroidJUnit4 (`androidx.test.ext:junit:1.3.0`) for instrumented tests
- Config: No separate `junit4` config file — test runner declared in `app/build.gradle.kts` via `testInstrumentationRunner = "androidx.test.runner.AndroidJUnitRunner"`

**Assertion Library:**
- `org.junit.Assert.*` (JUnit 4 built-in)

**UI Testing:**
- Compose UI Test (`androidx.compose.ui:ui-test-junit4`) declared in `androidTestImplementation`
- Espresso Core (`androidx.test.espresso:espresso-core:3.7.0`) declared but not yet used

**Run Commands:**
```bash
./gradlew test                    # Run local unit tests
./gradlew connectedAndroidTest    # Run instrumented tests (requires device/emulator)
./gradlew testDebugUnitTest       # Run debug unit tests only
```

## Test File Organization

**Location:**
- Unit tests: `app/src/test/java/com/example/vietforces/`
- Instrumented tests: `app/src/androidTest/java/com/example/vietforces/`
- Tests are **not co-located** with source files — they follow Android's standard source set separation

**Current Test Files:**
- `app/src/test/java/com/example/vietforces/ExampleUnitTest.kt` — placeholder unit test
- `app/src/androidTest/java/com/example/vietforces/ExampleInstrumentedTest.kt` — placeholder instrumented test

**Naming:**
- Test files should use `[SubjectClass]Test.kt` convention (e.g., `EloRankUtilsTest.kt`, `VocabularyRepositoryTest.kt`)
- Test methods: `camelCase` with snake_case for readability: `addition_isCorrect`, `useAppContext`

**Structure:**
```
app/src/
├── test/
│   └── java/com/example/vietforces/
│       └── ExampleUnitTest.kt        # local JVM tests (no Android framework)
└── androidTest/
    └── java/com/example/vietforces/
        └── ExampleInstrumentedTest.kt # device/emulator tests (Android context required)
```

## Test Structure

**Suite Organization (current boilerplate):**
```kotlin
// Unit test (no annotation needed):
class ExampleUnitTest {
    @Test
    fun addition_isCorrect() {
        assertEquals(4, 2 + 2)
    }
}

// Instrumented test:
@RunWith(AndroidJUnit4::class)
class ExampleInstrumentedTest {
    @Test
    fun useAppContext() {
        val appContext = InstrumentationRegistry.getInstrumentation().targetContext
        assertEquals("com.example.vietforces", appContext.packageName)
    }
}
```

**Patterns:**
- `@Test` annotation on each test method
- `@RunWith(AndroidJUnit4::class)` required on instrumented test classes
- No `@Before`/`@After` setup/teardown currently used
- Assertion via `assertEquals`, `assertTrue`, `assertFalse` from `org.junit.Assert.*`

## Mocking

**Framework:** None currently configured (no Mockito, MockK, or Turbine dependency)

**Current State:** No mocking is used — the two existing test files are placeholder stubs generated by Android Studio.

**Recommended Pattern for Future Tests:**
- Add MockK (`io.mockk:mockk`) for Kotlin-idiomatic mocking of singleton managers
- Singletons like `AiManager`, `UserProgressManager`, `PreferencesManager` would need to be refactored to interfaces or injected dependencies for testability
- Coroutine testing: add `kotlinx-coroutines-test` to `testImplementation` for `runTest {}` blocks

**What to Mock (when mocking is added):**
- `OpenAiClient` — network layer, avoid real HTTP in unit tests
- `PreferencesManager` — requires Android context; mock for unit tests
- AI responses — return canned `AiCallResult.Success(...)` or `AiCallResult.Error(...)` values

**What NOT to Mock:**
- Pure data models (`VocabularyItem`, `GameMode`, `EloRank`) — test directly
- `EloRankUtils` — pure functions, test with real inputs/outputs
- `VocabularyRepository` — pure in-memory list, test directly

## Test Types

**Unit Tests (`app/src/test/`):**
- Run on the local JVM — no Android framework dependency
- Suitable for: pure Kotlin logic, data model validation, utility functions
- Best candidates now: `EloRankUtils`, `VocabularyRepository` (pure functions), `AiManager` JSON parsers, `Screen` route construction

**Instrumented Tests (`app/src/androidTest/`):**
- Run on a connected Android device or emulator
- Require Android context for `PreferencesManager`, `SharedPreferences`, etc.
- Suitable for: `PreferencesManager` read/write, Compose UI interactions
- Compose UI tests via `createComposeRule()` from `ui-test-junit4`

**E2E Tests:**
- Not configured. Espresso is in `androidTestImplementation` but unused.

## Common Patterns

**Async Testing (recommended pattern when tests are written):**
```kotlin
import kotlinx.coroutines.test.runTest

class AiManagerTest {
    @Test
    fun gradeWriting_whenAiDisabled_returnsError() = runTest {
        // Requires kotlinx-coroutines-test dependency
        val result = AiManager.gradeWriting("Test topic", "some text")
        assertTrue(result is AiCallResult.Error)
    }
}
```

**Error Testing (recommended pattern):**
```kotlin
@Test
fun getCurrentRank_belowMinimum_returnsNewbie() {
    val rank = EloRankUtils.getCurrentRank(0)
    assertEquals("Newbie", rank.name)
}

@Test
fun getCurrentRank_atBoundary_returnsCorrectRank() {
    val rank = EloRankUtils.getCurrentRank(1200)
    assertEquals("Pupil", rank.name)
}
```

**Compose UI Testing (recommended pattern):**
```kotlin
@RunWith(AndroidJUnit4::class)
class MainScreenTest {
    @get:Rule
    val composeTestRule = createComposeRule()

    @Test
    fun mainScreen_showsVietforcesTitle() {
        composeTestRule.setContent {
            VietforcesTheme {
                MainScreen(
                    onProfileClick = {},
                    onNotificationClick = {},
                    onGameModeClick = {}
                )
            }
        }
        composeTestRule.onNodeWithText("Vietforces").assertIsDisplayed()
    }
}
```
